## 1. 개요 (Overview)

텔레그램 채널/봇으로 매일 올라오는 **주식 브리핑 메시지**를 자동으로 수집·구조화하여,
누구나 링크 하나로 **당일 시황 정리**와 **종목별 서머리(등락 이유 + 간단 차트)** 를
한눈에 확인할 수 있는 웹 서비스를 만든다.

텔레그램 메시지는 텍스트가 길고, 여러 종목이 뒤섞여 있어 원하는 종목만 빠르게 파악하기 어렵다.
Morning Brief는 이 메시지를 파싱·요약하여 **읽기 좋은 웹 페이지(공유 링크)** 로 변환한다.

### 핵심 가치 (One-liner)
> "텔레그램에 흩어진 주식 브리핑을, 링크 하나로 정리된 시황·종목 리포트로."

---

## 2. 배경 및 문제 정의 (Background & Problem)

### 현재 상황
- 리서치/트레이딩 데스크의 주식 브리핑이 텔레그램 채널에 텍스트로 게시된다.
- 메시지는 시황 코멘트 + 다수 종목 언급이 뒤섞인 긴 텍스트 형태다.
- 차트나 등락 이유는 별도 HTS/MTS/뉴스에서 다시 찾아봐야 한다.

### 문제점 (Pain Points)
| # | 문제 | 영향 |
| --- | --- | --- |
| P1 | 긴 텔레그램 텍스트에서 원하는 종목만 찾기 어렵다 | 정보 탐색 시간 증가 |
| P2 | 종목별 등락 이유가 흩어져 있거나 누락된다 | 판단 근거 부족 |
| P3 | 텍스트만으로는 가격 흐름(추세)이 직관적으로 안 보인다 | 시각적 맥락 부재 |
| P4 | 지난 브리핑을 다시 찾아보기 어렵다 (검색/보관 불가) | 히스토리 활용 불가 |
| P5 | 외부(고객/동료) 공유 시 텔레그램 원문을 그대로 전달해야 한다 | 가독성·전문성 저하 |

### 해결 방향
텔레그램 메시지를 **파싱 → 구조화(시황/종목/등락 이유) → 시각화(차트) → 공유 링크 발행**
하는 파이프라인을 구축한다.

## 3. 목표 (Goals & Non-Goals)

### 목표 (Goals)
- **G1.** 텔레그램 브리핑 메시지를 자동 수집한다.
- **G2.** 메시지에서 **시황 요약**과 **언급 종목 목록**을 추출한다.
- **G3.** 종목별로 **등락 이유(요약)** 와 **간단한 가격 차트**를 제공한다.
- **G4.** 하루 단위(또는 브리핑 단위)로 **공유 가능한 웹 링크**를 발행한다.
- **G5.** 과거 브리핑을 날짜별로 조회/검색할 수 있다.

### 비목표 (Non-Goals) — 최소 이번 범위(v1)에서 제외
- 매수/매도 등 **투자 추천·자동매매**는 제공하지 않는다.
- **실시간 틱 차트**나 초단위 스트리밍은 제공하지 않는다 (일/분봉 수준).
- 텔레그램 외 다른 소스(카톡, 이메일, 슬랙) 수집은 v1 범위 밖.
- 개인화된 포트폴리오 손익 관리 기능은 v1 범위 밖.

## 5. 사용자 시나리오 (User Scenarios)

1. **아침 시황 확인**
   장 시작 전, 텔레그램에 브리핑이 올라오면 봇이 자동으로 요약 링크를 함께 게시한다.
   사용자는 링크를 눌러 시황 한 줄 요약 + 오늘 언급된 종목 리스트를 본다.

2. **관심 종목 딥다이브**
   종목 리스트에서 "삼성전자"를 탭 → 등락 이유(예: "HBM 공급 확대 기대")와
   최근 5일 가격 차트를 확인한다.

3. **고객 공유**
   PB가 요약 링크를 고객에게 전달한다. 고객은 앱 설치 없이 웹에서 열람한다.

4. **과거 브리핑 회고**
   지난주 급등 종목의 그날 브리핑을 날짜로 검색해 다시 확인한다.

## 6. 기능 요구사항 (Functional Requirements)

### 6.1 텔레그램 메시지 수집 (Ingestion)
- **FR-1.1** 지정된 텔레그램 채널/봇의 신규 메시지를 자동 수신한다.
  - 방식: Telegram Bot API 웹훅(webhook) 또는 폴링, 필요 시 Telegram Client API(채널 구독).
- **FR-1.2** 원문 메시지, 작성 시각, 발신 채널, 첨부(이미지 등) 메타데이터를 원본 그대로 저장한다.
- **FR-1.3** 중복 메시지/수정 메시지를 식별해 재처리(idempotent)한다.

### 6.2 파싱 & 구조화 (Parsing / Structuring)
- **FR-2.1** 메시지에서 **시황 요약(market overview)** 문단을 식별·추출한다.
- **FR-2.2** 메시지에서 **언급 종목**을 추출하고 종목 마스터(종목코드/티커)와 매핑한다.
  - 종목명 표기 변형(예: "삼전", "삼성전자") 정규화.
- **FR-2.3** 종목별 **등락 방향(상승/하락/보합)** 과 **등락 이유(원인 문장)** 를 추출·요약한다.
  - 규칙 기반 + LLM 보조 요약(Claude API) 조합 권장.
- **FR-2.4** 파싱 결과에 **신뢰도/출처(원문 위치)** 를 함께 저장해 검증 가능하게 한다.

### 6.3 시세·차트 데이터 (Market Data)
- **FR-3.1** 매핑된 종목의 **가격 시계열**을 외부 시세 API에서 조회한다 (일봉 기준, 옵션으로 분봉).
- **FR-3.2** 종목별 **당일 등락률/종가/거래량** 등 핵심 지표를 함께 확보한다.
- **FR-3.3** 시세 조회 실패 시 차트 없이 텍스트 요약만 표시(graceful degradation).

### 6.4 시황 정리 페이지 (Market Summary Page)
- **FR-4.1** 브리핑 날짜, 시황 한 줄/문단 요약, 주요 지수(코스피/코스닥 등) 등락을 표시한다.
- **FR-4.2** 당일 언급 종목을 **등락률/이슈 순**으로 정렬한 리스트/카드로 제공한다.
- **FR-4.3** 각 종목 카드에서 종목 상세로 이동한다.

### 6.5 종목별 서머리 (Stock Summary) — **핵심**
- **FR-5.1** 종목명/코드, 현재가·등락률을 표시한다.
- **FR-5.2** **등락 이유**를 요약 문장(1~3줄)으로 표시하고, 원문(브리핑) 근거를 펼쳐볼 수 있게 한다.
- **FR-5.3** **간단한 차트**를 표시한다.
  - 기본: 최근 5~20영업일 종가 라인/캔들 차트.
  - 당일 등락 강조(상승 초록/하락 빨강 등 룰).
  - 모바일에서도 잘 보이는 경량 스파크라인 수준 이상.
- **FR-5.4** 관련 뉴스/링크가 있으면 함께 노출(옵션).

### 6.6 링크 발행 & 공유 (Link Publishing)
- **FR-6.1** 브리핑 단위(또는 일 단위)로 **고유 URL**을 발행한다. 예: `/brief/2026-07-08`
- **FR-6.2** 종목별 딥링크를 제공한다. 예: `/brief/2026-07-08/stock/005930`
- **FR-6.3** 발행된 링크를 텔레그램 채널에 자동 회신(선택) 또는 운영자가 공유한다.
- **FR-6.4** 링크는 로그인 없이 열람 가능(공개) 또는 토큰 기반 접근(비공개) 정책을 설정할 수 있다.

### 6.7 히스토리 & 검색 (History / Search)
- **FR-7.1** 날짜별 브리핑 아카이브를 제공한다.
- **FR-7.2** 종목명/키워드로 과거 브리핑을 검색한다.

## 8. 시스템 아키텍처 (Architecture)

```
[Telegram 채널/봇]
      │  (Webhook / Polling)
      ▼
┌──────────────┐    원문 저장     ┌──────────────┐
│  Ingestion    │ ───────────────▶│  Raw Store    │
│  Service      │                 │ (원문 메시지)  │
└──────┬───────┘                 └──────────────┘
       │ 신규 메시지 이벤트
       ▼
┌──────────────┐   종목/시황 추출  ┌──────────────┐
│  Parser &     │◀───────────────▶│  Stock Master │
│  Summarizer   │  (LLM: Claude)   │ (종목코드매핑) │
└──────┬───────┘                  └──────────────┘
       │ 종목 리스트
       ▼
┌──────────────┐   시세/차트 조회  ┌──────────────┐
│  Market Data  │◀───────────────▶│  시세 API     │
│  Fetcher      │                 │ (외부)        │
└──────┬───────┘                 └──────────────┘
       │ 구조화 결과
       ▼
┌──────────────┐                 ┌──────────────┐
│  Brief Store  │ ───────────────▶│  Web App /    │
│ (구조화 DB)   │   조회           │  Public Link  │
└──────────────┘                 └──────┬───────┘
                                        │ 링크 회신(옵션)
                                        ▼
                                 [Telegram 채널]
```

### 구성 요소
| 컴포넌트 | 역할 | 비고 |
| --- | --- | --- |
| Ingestion Service | 텔레그램 수신·원문 저장 | Bot API / Client API |
| Parser & Summarizer | 시황·종목·등락이유 추출/요약 | 규칙 + LLM(Claude API) |
| Stock Master | 종목명↔코드 매핑 정규화 | 사전/DB |
| Market Data Fetcher | 시세·차트 데이터 조회 | 외부 시세 API |
| Brief Store | 구조화 데이터 저장 | RDB(예: PostgreSQL) |
| Web App | 시황/종목 페이지 렌더 + 링크 발행 | SSR 권장(공유/SEO) |

```
Brief
  - id

  - source_channel

  - published_url
  - created_at

RawMessage
  - id
  - brief_id (FK)
  - raw_text
  - sent_at
  - attachments (JSON)

StockMention
  - id
  - brief_id (FK)
  - ticker                (예: 005930)
  - name
  - direction             (UP / DOWN / FLAT)
  - change_rate           (당일 등락률)
  - reason_summary        (등락 이유 요약)
  - reason_source_span    (원문 근거 위치/인용)
  - confidence

PriceSeries
  - id
  - ticker
  - interval              (1D / 1m 등)
  - points (JSON)         [{date, open, high, low, close, volume}, ...]
  - fetched_at
```

## 10. 비기능 요구사항 (Non-Functional Requirements)

| 구분 | 요구사항 |
| --- | --- |
| 성능 | 브리핑 수신 후 **5분 이내** 링크 발행. 페이지 로드 2초 이내(모바일 3G 기준 노력). |
| 가용성 | 장중 시간대 99.5% 이상 가용. 시세 API 실패 시 텍스트 폴백. |
| 확장성 | 채널 다중 구독, 일 수백 건 메시지 처리 가능. |
| 보안 | 텔레그램 토큰·API 키는 시크릿 저장소 관리. 비공개 링크는 토큰/만료 지원. |
| 정확성 | 종목 매핑 정확도 관리(오탐 시 운영자 보정 UI). LLM 요약은 원문 근거 병기. |
| 접근성 | 모바일 우선, 색맹 대비(등락 색+기호 병기), 반응형. |
| 규정 | 투자 자문이 아님을 명시하는 디스클레이머 노출. |

## 11. 성공 지표 (Success Metrics / KPI)

- **채택**: 발행 링크 대비 클릭률(CTR), 일간 열람 UV.
- **효율**: 브리핑 수신→링크 발행까지 평균 소요시간.
- **품질**: 종목 매핑 정확도, 등락 이유 요약 만족도(운영자/사용자 피드백).
- **재방문**: 과거 브리핑 검색·재방문율.
- **공유**: 링크 외부 공유 횟수.
